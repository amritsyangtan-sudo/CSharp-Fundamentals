# Generic Interfaces
If an interface uses `object` because it doesnot know the actual type, then a generic interface is ofter the better design.
Often but not always.


```csharp
    interface IRepository
    {
        void Add(object item);
        object Find(int id);

        Why object ?
            because interface is trying to work with Customer, Hall, Booking, Invoice etc.
    }

    class CustomerRepository : IRepository
    {
        public void Add(object item){}
        public object Find(int id){}
    }

    CustomerRepository customerRepository = new CustomerRepository();
    customerRepository.Add(new Hall());

    Issues: InvalidCastException. customerRepository is intended only for Customer not Hall. Run time error. not Compile time error.
----------------------------------------------------------------------------------------------------------

How Generic Interface solves the issues

    interface IRepository<T>
    {
        void Add(T item);
        T Find(int id);

        here the object
    }

    class CustomerRepository : IRepository<Customer>
    {
        public void Add(Customer customer){}
        public Customer Find(int id){}
    }

    Now compiler substituted T to Customer.
    Conceptually interface became
    interface IRepository
    {
        void Add(Customer item);
        Customer Find(int id);
    }

    if     
    CustomerRepository customerRepository = new CustomerRepository();
    customerRepository.Add(new Hall());  no compile. compile time error. The program doesnot run.
```
