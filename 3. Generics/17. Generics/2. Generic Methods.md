# Generic Methods
    A generic class makes the entire class reusable for different types
    If only methods need to work with different types then, this is where the generic methods is introduced.

# Scenario
```csharp
    suppose first requirment is print an integer
    class Printer
    {
        public void Print(int value) => Console.WriteLine(value);   works perfectly

        later string is required
        public void Print(string value) => Console.WriteLine(value);

        next print customer name
        public void Print(Customer customer) => Console.WriteLine(customer.Name);

        the methods in this class keeps growing.

        Problems:
            we are repeating the same logic for different types.

        Solution:
            Instead of public void Print(int value)
        
        we will write
        public void Print<T>(T value) => Console.WriteLine(value);

        Notice where <T> appears. In generic class it would have been in class Printer<T>
        for a generic method: void Print<T>(T value) 

        Now we can use:
        Printer printer = new Printer();
        printer.Print(10);
        printer.Print("Hello Generic Method");
        printer.Print(new Customer());
    }
```

# Generic Class vs Generic Method
**Generic Class:**
    whole class depends on the T. Everything in the class works with that type.

**Generic Method:**
    Only method depends on T. The rest of the class remains non-generic.
