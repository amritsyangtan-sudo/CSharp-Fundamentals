# Why Generic Class ?
    The class that uses a type parameter instead of a fixed type.

# The problem
## Example:
    Suppose we are workin gon Hall Boooking System.
    We need to create and reusable repository.
```csharp
    initially we only need customers.

    class CustomerRepository
    {
        public void Add(Customer customer) {}
        public Customer Find (int id) {}
    }

    Everything works. After some times wee need to create Hall Repository and BookingRepository
    we will copy everything like

    class HallRepository
    {
        public void Add(Hall hall){}
        public Hall Find(int id){}
    }

    class BookingRepository
    {
        public void Add(Booking booking){}
        public Booking Find(int id){}
    }

    Looking at the code. only one thing is changed. the word customer, hall, booking . everything is same.

    This is where Generics solves the problem. 

-----------------------------------------------------------------------------------------------
    Instead of class CustomerRepository{}, HallRepository {}, BookingRepository{}
    we will be doing

    class Repository<T>
    {
        public void Add(T item){}
        public T Find(int id){}
    }

    its usage: Repository<Customer> customers;  compiler now performs the substitution. T -----------> Customer
    Conceptually behind it becomes

    class Repository
    {
        public void Add(Customer item) {}
        public Customer Find(int id) {}
    }
    we never wrote this class but compiler substitue it.

    When we do Repository<Customer> repository;
    the compiler records Repository<T> then to ------------------> Repository<Customer>
    Now every T inside the class becomes Customer. Every field, method, property, parameter and return value.
 
    This is the reason why it is called Generic, which means general.
-----------------------------------------------------------------------------------------------
```

# Built in Generic Example.
```csharp
    List<int> numbers = new List<int>();
    List<string> names = new List<string>();
    List<Customer> customers = new List<Customer>();

    List is actually a generic class written by Microsoft.

    Conceptually(simplified) it looks something like this:

    public class List<T>
    {
        public void Add(T item){}
        public T Get(int index){}
        public void Remove(T item){}
    }

    Microsoft didnot wrote List<int> or List<string> different class for int and string. They wrote one class List<T>
```
