# Why Generic Class ?
    The class that uses a type parameter instead of a fixed type.

# The problem
## Example:
    Suppose we are workin gon Hall Boooking System.
    We need to create and reusable repository.
```csharp
    initially we only need customers.

    class CustomerRepository
    {
        public void Add(Customer customer) {}
        public Customer Find (int id) {}
    }

    Everything works. After some times wee need to create Hall Repository and BookingRepository
    we will copy everything like

    class HallRepository
    {
        public void Add(Hall hall){}
        public Hall Find(int id){}
    }

    class BookingRepository
    {
        public void Add(Booking booking){}
        public Booking Find(int id){}
    }

    Looking at the code. only one thing is changed. the word customer, hall, booking . everything is same.

    This is where Generics solves the problem.
```
