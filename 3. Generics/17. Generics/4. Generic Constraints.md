# The Problem
```csharp

so far we are able to do
class Repository<T>
{
    void Add(T item){}
}

Here, T can be any type.


First Problem
class Factory<T>
{
    public T Create()
    {
        return new T();      //error Cannot create an instance of the variable type 'T' because it does not have the new() constraint
    }
}


class Factory<T>
{
    public T Create(T value)
    {
        return value;       // no error. object was never created. 
    }
}


What happens in compiler.
When compiler sees Factory<T> it know it only know it is some type.
Compiler doesnot know whether it is customer, hall, booking, invoice etc


Solution:
class Factory<T> where T : new()
{
    public T Create()
    {
        return new T();
    }
}


class Factory<T>
{
    public T Create(T value)
    {
        return value;
    }
}
```



# Uses
A generic constraint tells a compiler which type of `types` are allowed to repalce `T`.
C# generic allows methods, interface, class to work with any data type. sometimes, we want to 
restrict the types of argument that can be passed to the generic type.

Without constraint Generic can accept any type which avoids type safety.

# Types of constraint
- where T : struct (Restrict T to value types int, double , bool or user defined struct). cannot be null
- where T : class (Restrict T to refernce types(class, interface, delegate, array))
- where T : new() (Restrict T to types that have a public parameterless constructor)
- where T : InterfaceName (Restrict T to types that implements a specific interface)'
- multiple Constraints
    where T : class, new()
- multipel type Parameters
    class Sample<T1, T2> where T1: class Where T2: struct

# Why to use Generic Constraint
- Restrict which types can be used. Only defined/valid one are allowed.
- Tell compiler what T can do.
- Allow object creation new T()
- Restrict to value types or reference types
