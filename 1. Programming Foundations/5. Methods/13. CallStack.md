# Call stack
**Suppose we Wrote**
    
```csharp
    static void Main()
    {
        SayHello();
    }

    static void SayHello()
    {
        Console.WriteLine("Hello");
    }
    
    Call stack is a special memory structure that keeps track of active method calls.

    Program Starts:
    - CLR creates first stack frame
    - Main becomes the active method.
    - Main reaches SayHello();
    - CLR creates another stack frame.
            CALL STACK
                +-------------------+
                | SayHello()        | ← executing
                +-------------------+
                | Main()            |
                +-------------------+
    - SayHello() executes Console.WriteLine("Hello");
            CALL STACK
                +---------------------------+
                | Console.WriteLine()        | ← executing
                +---------------------------+
                | SayHello()                 |
                +---------------------------+
                | Main()                     |
                +---------------------------+
    - Console.WriteLine("Hello"); finishes
            CALL STACK
                +-------------------+
                | SayHello()        |
                +-------------------+
                | Main()            |
                +-------------------+    
    - SayHello() finishes
            CALL STACK
                +-------------------+
                | Main()            |
                +-------------------+
    - Main Finishes
            CALL STACK
                (empty)

```

### What is Stack Frame ?
    Stack Frame is one method invocation. 
```
Every frame contains informatin like
|-------------------------------|
| Parameters                    |
| Local Variables               |
| Return Address                |
| Temporary Values              |
|-------------------------------| 

Evey stack frame stores something called return address.
CPU contains a special register called **Stack Pointer**
    Calling a method pushes a new frame onto the stack.
    Returning from a method popos that frame off.
    The Stack Pointer always points to the current top of the stack.


The Call Stack is the computer's way to remember where it is, where it came from, and where it must return.
```
