# Method Overloading
    Multiple methods can have same name, as long as the parameter signatures are different.

# Examples
```csharp
    void Print(string message) => Console.WriteLine(message);
    void Print(int number) => Console.WriteLine(number);
    void Print(bool value) => Console.WriteLine(value);

    All three methods are named Print but each accepts different parameter type.

    Why doesnot compiler gets confused ?
    Print(10)   how does the compiler knows which method to execute ?
            It examines the argumets and find the matching method.
    
    Compiler identify a method using its method signature.
    Method signature consists of
        - Method name
        - Number of parameters
        - Parameter types
        - Parameter order

    ** Return type and parameter names are not included
       ** int Add(int a, int b) =>  a + b; **
        ** double Add(int a, int b) => a + b **
        Error : compiler doesnot consider return type when overloading.


    Imagine a compiler has a table like this
    Print()
        ├───────────────┐
        │ Print(string) │
        ├───────────────┤
        │ Print(int)    │
        ├───────────────┤
        │ Print(bool)   │
        └───────────────┘
    
```
