Example:
    void IncreaseAge(int age)
    {
        age++;
    }

    int age = 20;
    Increase(age);
    Console.WriteLine(age);

    Expectation:
        21
    But Result is: 20


Explanation:
    int age = 20;       ---> memory address(1000) age = 20 (there is one variable named age inside main)
    Now we write IncreaseAge(age);  ----> it didnot copy the variable, it only copy the value of the main.age variable value into new age variable.
    Both are named aged and they live in different stack frames like main.age and method.age
    when IncreaseAge(age) is invoked. it changes the age of address 10001 not 1000

Now Using Ref
    void IncreaseAge(ref int age)   using ref is not passing the value of age but location of the variable
    {
        age++;
    }

    int age = 20;
    IncreaseAge(ref age);
    Console.WriteLine(age);

    Output: 21

Case: 1
    int age = 20;
    IncreaseAge(ref number);  it will throw error since number isnot defined anywhere
    static void IncreaseAge(ref int age)
    {
        age++;
    }

Case: 2
    int age = 20;
    int number = 50;

    IncreaseAge(ref number);

    Console.WriteLine(age);
    Console.WriteLine(number);

    static void IncreaseAge(ref int age)    parameter is named age but number is passed as argument.
    {
        age++;      even though it is named age. it is modifying the number.
    }

    Output:
        age will be 20
        number will be 51


Lesson:
    A variable is not identified by its name, it is identified by its storage location in memory
