# Why Properties?
Classes needed a way to control read or change of data through rules. instead of exposing the variable itself the class
exposes controlled access which is called **property**

`Property` is a member that controls access to data. `Property` itself is not the data. Data is stored somewhere else (typically in field).

```csharp
    Instead of 
    public string Name;

    we write:
    public string Name {get;set;}  This is auto implemented property.

    Behind the scenes the compiler transform this code  public string Name {get;set;} to 
  
    private string _name;
    public string Name
    {
        get
        {
            return _name;
        }
        set
        {
            _name = value;
        }
    }

    we didn't wrote _name but compiler did.

```

# Why use Properties

```csharp
 customer.Name = "";  we are allowed to pass the empty name, names cannot be empty.

 With property

 private string _name;
 public string Name
 {
    get
    {
        return _name;
    }
    set
    {
        if(string.IsNullOrWhiteSpace(value))
            throw new ArgumentException();
        _name = value;
    }
 }

 now the class protects itself.

```
